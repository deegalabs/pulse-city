## Three message pillars

Exactly three. Each one binds to an owned whitespace phrase, each one has proofs Lia can verify, and each one produces real surface strings.

### Pillar 1 — One primitive, two tempos
**Owned phrase:** *the studio is the radio, paused.*

**1-line claim.** Composition and listening are the same act at different tempos, sharing one URL scheme.

**Proofs:**
1. A /radio/[name] station is composed of the same pattern objects /studio edits — no conversion, no export.
2. /p/[id] shows the pattern code *and* plays it; the "open in studio" button is one click.
3. The player is a single monorepo package (`packages/pulse-player`) — not four products pretending to be one.

**Real surface strings:**
- Marketing hero: `the studio is the radio, paused. any hand can pick it back up.`
- /radio footer: `radio/ipe is a /studio pattern, slowed to wall-clock time. open it in /studio to see how.`

### Pillar 2 — Readable is alive
**Owned phrase:** *visible craft at ambient tempo.*

**1-line claim.** A soundtrack you can read is one you can keep — which is what makes it alive instead of finished.

**Proofs:**
1. Every /p/[id] page has the pattern code above the fold. No "view source," no modal.
2. Agent-written patterns come with visible diffs and the agent's MCP key, not a "generated by AI" badge.
3. JetBrains Mono ships as a local font. The code rendering is not a screenshot; it is the code.

**Real surface strings:**
- /p/[id] headline: `"slow kitchen" by lia. the code is on the page. open in studio, or just listen.`
- Docs intro: `patterns are text. if you can read text, you can fork the soundtrack.`

### Pillar 3 — Shared, not personalized
**Owned phrase:** *shared soundtrack for a pop-up room.*

**1-line claim.** The room hears one song at one moment; the place remembers you were here because the place was playing for everyone, including you.

**Proofs:**
1. /radio/[name] is wall-clock synced — the kitchen and the coworking floor hear the same bar at the same second.
2. There is no recommendation engine, no "for you," no per-listener mix.
3. `<pulse-widget>` embeds carry the author credit with them; a host site cannot strip it without modifying the AGPL source.

**Real surface strings:**
- /radio/ipe hero: `radio/ipe is playing "slow kitchen". 04:11 until the next pattern. same song for the kitchen, the coworking floor, and you.`
- Embed tooltip: `playing on pulse.city — "slow kitchen" by lia. shared with this room.`

## Naming rules

### Product name
- **Canonical prose form:** `pulse.city` (lowercase, dot, no hyphen, no camel).
- **Wordmark display form:** `PULSE·CITY` (Chakra Petch, sky `#3aa5ff` middot, uppercase).
- **Forbidden forms:** `pulse-city` (except in package/repo slugs like `pulse-city-next`), `PulseCity`, `Pulse.City`, `Pulse City`, `pulse_city`.
- **Rationale:** Lia copy-pastes the URL into a Telegram group chat. The dot has to survive that trip. The wordmark lives in marks only; the dot lives in prose.

### Pattern titles
- **Rule:** lowercase, quoted in copy, verb-or-noun phrases are both fine, max ~32 characters.
- **Examples:** `"slow kitchen"` · `"monday standup loop"` · `"still here at 3am"`.
- **Forbidden:** Title Case (`"Slow Kitchen"`), exclamation marks (`"slow kitchen!"`), emoji in titles, version numbers (`"slow kitchen v2"` — fork it instead).

### Radio stations
- **Slug rule:** `radio/{slug}` — lowercase, kebab-case, ASCII only, max 24 characters.
- **First station:** `radio/ipe` (Ipê Village 2026).
- **Prose form:** `radio/ipe` in running text, `radio/ipe` in UI (no capital R, no slash-free form).
- **Forbidden:** `Radio/Ipe`, `/radio/IPE`, `radio/ipê` (ASCII only for slugs; display names can carry accents).
- **Display name** (what renders on the station page) is separate from the slug and may use accents, spaces, and sentence case: `radio/ipe` → display `"ipê village 2026"`.

### Share URLs
- **Scheme:** `/p/{id}`.
- **id format:** base62, 8 characters minimum, 12 maximum. Short enough to fit in a tweet and a printed flyer, long enough to never collide for the Ipê Village month.
- **OG image:** uses the pattern's own punchcard render, not a stock hero. This is Creator + Sage in one image — the work is its own cover art.
- **Forbidden:** long hashes, sequential integers, user-chosen slugs (preserves a clean namespace; display titles cover the "name it" need).

### MCP keys
- **Prefix:** `pc_` (GitHub-PAT style, already decided in use-cases.md, locked here).
- **Scope:** per-user, not per-agent-family. One key ↔ one author identity. An agent using Lia's key composes *as* Lia until revoked.
- **Display rule:** never print the full key in UI after issuance; show `pc_****` + last 4 characters.
