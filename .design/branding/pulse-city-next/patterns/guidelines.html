logo

A 3x7 punchcard. Twenty-one dots, one saturated. The center dot is listener sky -- the only bright dot in the grid. The mark says awake, and remembers you were here: still, but with one bright acknowledgement.

rules

  • Clear space: 1u = one dot cell on all sides.
  • Row 1 and Row 3 are all text-dim (#64748b). Row 2 is text (#e2e8f0) with column 4 in listener (#3aa5ff).
  • Below 16px bounding height: use cropped 5x3 favicon variant or ASCII [...o...].
  • Never rotate, skew, recolor, animate the outer dots, or embed in gradients.
  • The center dot is the only saturated dot. Never duplicate saturation elsewhere in the grid.
  • Prose form is pulse.city (lowercase, period). Display form is PULSE·CITY (middot, Chakra Petch 700).

positioning

pulse.city is the living soundtrack primitive for places that want their music as readable as their code -- one URL scheme where the studio is the radio, paused, and any hand (human or agent) can pick it back up.

whitespace claims

  • The studio is the radio, paused. One surface, two modes. You never "publish." You stop, and the city keeps playing what you left it.
  • Visible craft at ambient tempo. The code is on the screen because the code is the song. No hidden state, no black box.
  • Shared soundtrack for a pop-up room. Not personalized, not a playlist. A place has one pulse, and anyone can shape it.

hard rules

  • No gradients anywhere -- not hero, not buttons, not chrome.
  • No pastel tints. No creator-100, no listener-50.
  • No second border color. One value, period.
  • Page background floor: #0a0e17. Nothing darker. Surface ramps go up.
  • Max one signal-warn firing per screen. Amber is precious.
  • Agent never renders as long-form body text on surface-1 (3.6:1). Chrome only.

pairing notes

  • Chakra Petch at display + heading tiers, DM Sans at heading-sm and below. The crossover is at 1.25rem.
  • JetBrains Mono for inline code, code blocks, and any numeric data (BPM, timestamps, counts).
  • Space Mono exclusively for micro-labels and bracketed buttons. Always uppercase, always tracked +0.12em. Never below 0.6875rem.
  • Fluid clamp() on display and heading tiers only (375px to 1440px). Body tiers are fixed.

In production, these fonts load via next/font/local with WOFF2 from src/app/fonts/. The Google Fonts CDN in this guide is for preview only.

visual elements

Two motion shapes. One static dither. One icon library. No stock photos. No aesthetic code backdrops. The imagery comes from @strudel/draw output -- the music visualizes itself.

Nothing else moves. No hover bounces. No scroll parallax. No page-load fades. The brand is still, except when something happened.

iconography -- Lucide

Library: Lucide (ISC, AGPL-compatible). Stroke 1.5px on 24px canvas -- slightly thinner than default 2px, matching the understated voice. Outline only, no filled variants. Color inherits from currentColor. Sizes: 16 / 20 / 24 / 32.

voice

Present · Legible · Understated. Confidence without volume. The product is loud; the voice is not.

three voice words

voice 01
present

current tense. "the city is playing." not "will play." not "could play." describe the state, don't sell the feature.

voice 02
legible

real numbers or nothing. no jargon unless the jargon is the point. code is OK; "synergy" is not.

voice 03
understated

lowercase by default. UPPERCASE for /studio micro-labels only. never exclaim. not once. name the actor.

seven voice rules

  1. Describe the state, don't sell the feature. "3 listeners" not "join the community."
  2. Present tense, no futures. "the station plays" not "will play." Real numbers or nothing.
  3. Lowercase by default. UPPERCASE for /studio micro-labels only ([ SAVE ], PLAYING).
  4. Never exclaim. Not once. Not even on success. Not even in error messages.
  5. Name the actor. "agent evolved pattern at 03:21" not "pattern was evolved."
  6. Errors: state the fact plainly, provide the next step, no apology theater. [ERR] pattern did not parse -- line 3
  7. Code blocks are part of the copy. A marketing page can show 4 lines of real Strudel and that IS the pitch.

applications

Same brand, different volume. The hard rule: ration the loud. /studio shouts in functional chrome. /radio barely whispers. /p is editorial. /embed is a guest.

surface volume dominant accent type dominance dither bracketed buttons
/studio loud creator (lime) mono, micro 2-4% everywhere
/radio quiet listener (sky) body, display 0% [ PLAY ] only
/p editorial listener (sky) display, body, code minimal none
/embed minimal listener (sky) body, micro 0% [ PLAY ] only
/ marketing balanced listener + creator display, body hero 4% [ LISTEN ]
